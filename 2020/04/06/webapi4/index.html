事件监听：

事件监听注册事件

语法：事件源.addEventListener(‘事件类型’,事件处理程序,是否捕获);

  1. 事件源 → 操作的元素
  2. 事件类型→ 在此不加on, onclick→click
  3. 事件处理程序→ 函数
  4. 是否捕获，可选参数，默认为false。 true→ 捕获，false→冒泡

事件监听移除事件

语法：元素.removeEventListener(‘事件类型’，事件处理程序名称);

  • 事件源，操作的元素
  • 事件类型，不加on
  • 事件处理程序的名字：函数名
    • 注意：若将来要通过事件监听的方式移除事件时，之前注册事件要把事件处理程序单独抽取出来命名 。

总结：

​ 注册事件：

​ 1、元素.onXXX = function () {}

​ 2、元素.addEventListener(‘事件类型不加on’,function () {});

​ 移除事件：

​ 1、元素.onXXX = null;

​ 2、元素.removeEventListener(‘事件类型不加on’,事件处理成的名称);

事件：

键盘事件

  • 事件类型
    • onkeydown 键盘按下事件
    • onkeyup 键盘弹起事件
    • 注意：一般情况给document绑定键盘事件。
  • 鼠标事件：
    • oninput
    • onmousedown
    • onmouseup
    • onmousemove
    • onmouseover：onmouseenter
    • onmouseout：onmouseleave
    • onmousover：支持事件冒泡
    • onmouseenter：不支持事件冒泡

键盘事件对象相关属性

键盘事件对象相关属性-键盘按下哪个键

  • ==事件对象.keyCode== 获取键盘按键对应的键码值
  • 事件对象.altKey 表示alt键是否按下，返回布尔值。
  • 事件对象.shiftKey 表示shift键是否按下，返回布尔值。
  • ==事件对象.ctrlKey== 表示ctrl键是否按下，返回布尔值。（true按下，false没有按下）

事件对象的公共属性和方法

  • ==事件对象.target== 获取最先触发的元素
  • 和this的区别
    • this在事件处理程序中始终代表的是事件源
    • e.target 代表不一定是事件源，代表的是最先触发的元素【目标阶段→ document】

方法：

  • 事件对象.preventDefault(); 阻止默认行为
  • ==事件对象.stopPropagation();== 停止冒泡传播
  • 【onmouseover支持冒泡，onmouseenter不支持冒泡】

课程回顾：

​ 删除元素：removeChild

​ 插入元素：inserBefore，替换元素：replaceChild，克隆元素：cloneNode

​ 事件注册：

​ 1、元素.onXXX = function () {}

​ 2、元素.addEventListener(‘事件类型不加on’,function () {代码},捕获);

​ 事件移除：

​ 1、元素.onXXX = null;

​ 2、元素.removeEventListener(‘事件类型不加on’,处理程序名称);

​ 事件流：事件触发后三个流程：捕获，目标，冒泡

​ 事件对象：事件发生后的相关信息对象;事件源.事件类型 = function (e) {第一个形参就是事件对象}

​ 获取鼠标信息：e.clientX，e.clientY，e.pageX，e.pageY，e.offsetX，e.offsetY

​ 事件：键盘（onkeydown，onkeyup），鼠标（onmousedown，onmouseup，onmousemove）

​ onmouseover（支持事件冒泡）：onmouseenter（不支持事件冒泡）

​ onmouseout：onmouseleave

​ 键盘信息：事件对象.keyCode

​ 事件对象属性：事件对象.target【最先触发事件的元素】

​ 方法：事件对象.preventDefault(); 阻止默认行为==事件对象.stopPropagation();== 停止冒泡传播
